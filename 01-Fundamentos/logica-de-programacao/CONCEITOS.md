## Conceitos fundamentais

### 1. Algoritmo

Um algoritmo é uma sequência finita de passos para resolver um problema. Pode ser comparado a uma receita de bolo: cada passo deve ser bem definido para chegar ao resultado desejado.

Exemplo simples de algoritmo:

1. Ler o número de um usuário.
2. Verificar se o número é maior que zero.
3. Se for maior, mostrar "Número positivo".
4. Se não for, mostrar "Número negativo ou zero".

### 2. Variáveis

Variáveis são espaços na memória utilizados para armazenar dados. Elas recebem nomes e guardam valores que podem mudar durante a execução do programa.

Exemplo:

- nome = "Ana"
- idade = 20

### 3. Estruturas de decisão

As estruturas de decisão permitem que o programa escolha um caminho diferente dependendo de uma condição.

Exemplo:

- se a idade for maior ou igual a 18, mostrar "Maior de idade";
- caso contrário, mostrar "Menor de idade".

Essas estruturas são fundamentais para criar programas que tomam decisões.

### 4. Estruturas de repetição

As estruturas de repetição permitem executar um bloco de código várias vezes, sem precisar escrever o mesmo trecho várias vezes.

Exemplo:

- repetir uma ação enquanto uma condição for verdadeira;
- repetir uma ação um número determinado de vezes.

Elas são muito úteis em tarefas como percorrer listas, validar dados e automatizar processos.

### 5. Funções e procedimentos

Funções e procedimentos ajudam a organizar o código em blocos menores e reutilizáveis. Em vez de repetir a mesma lógica várias vezes, ela pode ser agrupada em uma função e chamada quando necessário.

Isso torna o programa mais legível, organizado e mais fácil de manter.

## Como pensar logicamente

Pensar logicamente envolve quebrar um problema em partes menores e entender claramente o que precisa ser feito. Alguns passos importantes são:

1. Entender o problema.
2. Identificar as entradas e saídas.
3. Definir os passos necessários para chegar à solução.
4. Organizar os passos em ordem.
5. Testar e corrigir possíveis erros.

Esse processo é semelhante ao modo como um engenheiro resolve um problema: primeiro compreende, depois planeja e só então executa.

## Pseudocódigo

O pseudocódigo é uma forma de representar a lógica de um algoritmo sem depender da sintaxe de uma linguagem específica. Ele é usado para facilitar o entendimento do raciocínio antes de escrever o código real.

Exemplo de pseudocódigo:

```text
inicio
    ler numero
    se numero > 0 então
        mostrar "positivo"
    senão
        mostrar "negativo ou zero"
fim
```

O pseudocódigo ajuda a pensar de forma mais clara e evitar erros de implementação.
